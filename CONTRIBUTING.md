# Contributing and Update Rules

## Goal

- Keep the knowledge base clinically practical, easy to scan, and safe to update over time.

## Core Principles

- Prefer concise, high-yield summaries over textbook-length prose.
- Use the existing template whenever creating a new topic.
- Write for bedside use first, teaching use second.
- When medication doses, thresholds, or algorithms are included, add a source and update date.

## File Organization

- `01_危急處置/` for unstable or immediately life-threatening conditions.
- `02_主訴導向/` for symptom-first bedside approaches.
- `03_疾病主題/` for diagnosis-centered pages.
- `04_流程與檢查/` for evaluation workflows and diagnostic tools.
- `05_藥物/` for medication summaries.
- `06_程序/` for procedures and preparation guides.
- `07_附錄/` for scores, abbreviations, and references.
- `08_ACLS_2025/` for ACLS-specific modules.

## Writing Standard

- Start with `一句話重點`.
- Include `立即危險訊號`, `初始評估`, `建議檢查`, `急診處置`, and `Disposition`.
- Use short bullets rather than long paragraphs whenever possible.
- Add `院內流程` if local practice differs from generic guidance.

## Update Standard

- If you change a management recommendation, review the whole page for consistency.
- If you add dose-based content, verify against the newest trusted source before clinical use.
- If a topic becomes too broad, split it into a symptom page and a disease page.

## Git Workflow

- Pull before major edits if working on multiple computers.
- Use clear commit messages such as `Add bowel obstruction topic` or `Update ACLS README`.
- Push often enough to avoid losing incremental knowledge-base changes.

## Clinical Safety Note

- This repository is a decision-support aid, not a replacement for official guidelines, hospital policy, or specialist judgment.
